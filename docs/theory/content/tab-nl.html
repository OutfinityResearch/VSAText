NL Tab (Natural Language)

The NL (Natural Language) tab generates prose narrative from your CNL specification. This is where your story blueprint transforms into readable fiction.

When to Use This Tab

Use the NL tab after you've defined your story elements and structure. This tab produces the actual story text based on your specification.

What This Tab Does

The NL tab takes your complete story specification and generates readable narrative prose. It respects:

  • Character traits and archetypes for voice and behavior
  • Location atmospheres for scene descriptions
  • Mood definitions for emotional tone
  • Relationship dynamics for dialogue and interaction
  • World rules for consistency
  • Narrative arc for pacing

Generation Options

Option Description Values
Strategy Generation approach Random (fast), LLM (quality), Advanced (combined)
Scope What to generate Full story, specific chapter, single scene
Detail Level Prose density Summary, Standard, Detailed
Style Literary voice Literary, Commercial, Minimalist

Generation Strategies

Random Strategy

Fast, deterministic generation using template-based prose. Good for quick previews and iteration.

LLM Strategy

Uses language models for richer, more natural prose. Requires API configuration.

Advanced Strategy

Multi-pass generation with planning, drafting, and revision phases. Highest quality but slowest.

Output Display

The generated prose appears with:

  • Word Count: Total words generated
  • Reading Time: Estimated reading duration
  • Quality Score: NQS (Narrative Quality Score) of output
  • Export Options: Download as TXT, DOCX, or HTML

Example Workflow

  1. Define characters, locations, and relationships in respective tabs
  2. Set up story structure (books, chapters, scenes)
  3. Choose narrative arc and map beats in Blueprint
  4. Select moods for key scenes
  5. Navigate to NL tab
  6. Choose generation strategy and scope
  7. Click "Generate Story"
  8. Review output, refine specification, regenerate as needed

Quality Considerations

Generation quality depends on specification completeness:

Specification Quality Generation Result
Minimal (just characters) Generic, template-heavy prose
Basic (characters + structure) Coherent but predictable narrative
Complete (all elements defined) Rich, consistent, character-driven story

Metrics Impact

The NL output is evaluated against your specification:

  • Character Consistency (CS): Do characters behave according to traits?
  • Location Logic (locLogic): Are settings used coherently?
  • Relationship Usage (relUsage): Do defined relationships appear?
  • Emotional Arc Profile (EAP): Does pacing match target arc?
