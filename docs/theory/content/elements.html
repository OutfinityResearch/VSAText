Story Elements Guide

This document provides detailed documentation for each element type available in SCRIPTA's visual editor. Each section corresponds to a tab in the Story Forge interface.

Editor Tabs Overview

SCRIPTA organizes story elements into specialized tabs for focused editing. Each tab manages a specific aspect of your story specification, from characters and locations to emotional arcs and dialogue planning.

Characters Tab

The Characters tab manages all persons, creatures, and entities that act within your story.

Character Properties

Property Description Values
Name Character's identifier Any string
Type Role in the narrative protagonist, antagonist, mentor, ally, character
Archetype Jungian/Campbell archetype Hero, Shadow, Mentor, Herald, Threshold Guardian, Shapeshifter, Trickster, Ally
Traits Personality characteristics courage, wisdom, cunning, loyalty, ambition, compassion, etc.
Arc Type Character's transformation positive (grows), negative (corrupts), flat (inspires others)
Flaws Weaknesses for conflict fears failure, arrogance, naivety, etc.

CNL Output

Anna is protagonist
Anna has archetype Hero
Anna has trait courage
Anna has trait determination
Anna has flaw "fears failure"
Anna has arc positive

For complete character theory, see Characters Documentation.

Relations Tab

The Relations tab defines connections between characters. Relationships affect how characters interact and drive plot dynamics.

Relationship Types

Category Types Story Impact
Family sibling, parent_child, spouse, family Creates inherent loyalty/conflict, inheritance plots
Friendship friend, ally, companion Support network, sacrifice opportunities
Conflict enemy, rival, nemesis Direct opposition, competitive tension
Hierarchy mentor_student, leader_follower, master_servant Knowledge transfer, power dynamics
Romantic romantic, former_lovers, betrothed Emotional stakes, subplot fuel
Betrayal betrayer, victim_aggressor Trust violations, revenge arcs

Relationship Properties

  • Source: The character from whose perspective the relationship is defined
  • Target: The other character in the relationship
  • Type: The nature of the connection
  • Dynamics: How the relationship evolves (optional)

CNL Output

Anna relates to Marcus as sibling
Anna relates to Gandalf as mentor_student
Marcus relates to Drakon as enemy

Metrics Impact

Relationships affect the Relationship Usage Score (relUsage). Defined relationships should manifest in the narrative - characters in a relationship should appear together in scenes. Low relUsage indicates "decorative" relationships that don't impact the story.

Locations Tab

The Locations tab manages places where scenes occur. Well-defined locations enhance atmosphere and spatial coherence.

Location Properties

Property Description Examples
Name Location identifier Village, Dark Forest, Castle
Geography Physical type urban, rural, forest, mountain, coastal, desert, space
Atmosphere Emotional quality peaceful, mysterious, threatening, welcoming, desolate
Time Period Historical setting ancient, medieval, modern, futuristic
Characteristics Defining features "ancient stone walls", "bioluminescent plants"

CNL Output

Village is location
Village has geography rural
Village has atmosphere peaceful
Village has time_period medieval
Village has characteristic "ancient stone walls"
Village has characteristic "central fountain"

Metrics Impact

Locations affect the Location Logic Score (locLogic). Good stories reuse key locations rather than introducing a new location for every scene. Low locLogic indicates "random scene-hopping" without spatial coherence.

Objects Tab

The Objects tab (also called Plot Elements) manages physical items, artifacts, and documents that drive the plot.

Object Types

Type Description Examples
Artifact Magical or significant item Ring of Power, Horcruxes, Excalibur
Key Unlocks access or secrets Ancient key, password, map
Document Contains information Letter, prophecy, will, journal
Weapon Tool for conflict Sword, gun, poison
MacGuffin Drives plot but content unimportant Briefcase in Pulp Fiction, Maltese Falcon
Symbol Represents theme Mockingbird, Green Light, Rosebud

Object Properties

  • Significance: How important to the plot (high, medium, low)
  • Owner: Who possesses the object
  • Description: What it looks like and does
  • First Appearance: When introduced (for Chekhov's Gun)

CNL Output

SilverKey is artifact
SilverKey has significance high
SilverKey has description "Ancient key that unlocks the forbidden chamber"
Anna owns SilverKey

For plot element theory, see Plot Elements Documentation.

Moods Tab

The Moods tab defines emotional atmospheres for scenes. Moods combine multiple emotions with varying intensities.

Preset Moods

Mood Primary Emotions Best For
Mysterious curiosity, unease, wonder Discovery scenes, hidden secrets
Ominous fear, anxiety, tension Approaching danger, foreshadowing
Triumphant joy, pride, hope Victories, achievements
Melancholic sadness, nostalgia, longing Loss, memories, bittersweet moments
Romantic love, longing, hope Love scenes, emotional connections
Tense tension, anxiety, fear Confrontations, chase scenes
Peaceful serenity, gratitude, hope Rest points, ordinary world
Adventurous curiosity, wonder, hope Journey beginnings, exploration
Horrific fear, disgust, despair Horror reveals, monster encounters
Epic awe, tension, hope Climactic battles, grand reveals
Bittersweet joy, sadness, nostalgia Endings, farewells, sacrifice

Custom Moods

You can create custom moods by combining base emotions with intensity weights (1-3):

CustomMood is mood
CustomMood has emotion curiosity 2
CustomMood has emotion fear 1
CustomMood has emotion wonder 3

Metrics Impact

Moods affect the Emotional Arc Profile (EAP). The sequence of moods across scenes should align with the chosen narrative arc to create proper pacing.

Dialogues Tab

The Dialogues tab structures conversations between characters. Dialogues in SCRIPTA are specifications, not full scripts.

Dialogue Properties

Property Description Options
Purpose Why this conversation happens reveal, conflict, bonding, exposition, negotiation, confession, argument, farewell
Participants Characters involved List of characters with roles (speaker/listener)
Tone Emotional register confrontational, intimate, formal, casual, tense
Tension Conflict level (1-5) 1 = calm, 5 = explosive
Beat Link Associated narrative beat Any beat from selected arc

Dialogue Exchanges

Each dialogue contains exchanges - outline entries for what each character conveys:

  • Speaker: Who is speaking
  • Intent: What they want to convey
  • Emotion: How they feel
  • Sketch: Rough version of the line

Metrics Impact

Dialogues affect the Dialogue Quality Score (dialogueQuality). Well-defined dialogues have purpose, multiple participants, and substantive exchanges. Empty or placeholder dialogues reduce this score.

Blocks Tab

The Blocks tab manages reusable narrative building blocks - predefined templates for common story moments.

Block Categories

Category Blocks
Revelation secret_revealed, identity_exposed, truth_discovered, prophecy_fulfilled
Conflict confrontation, battle, argument, chase, standoff
Transformation death_rebirth, sacrifice, redemption, corruption, enlightenment
Journey crossing_threshold, tests_allies, approach_cave, return
Relationship reunion, betrayal, rescue, farewell, alliance
Decision choice, temptation, refusal, acceptance, dilemma

Using Blocks

Blocks can be assigned to scenes to indicate what type of narrative moment occurs. This helps with:

  • Ensuring variety of scene types
  • Aligning scenes with arc beats
  • Generating appropriate content

Blueprint Tab

The Blueprint tab provides high-level story planning tools that connect abstract structure to concrete content.

Blueprint Components

1. Arc Selection

Choose the narrative arc template that guides your story's structure.

2. Beat Mapping

Assign arc beats to specific chapters and scenes. Each mapping includes:

  • Beat: The narrative beat (e.g., "Call to Adventure")
  • Chapter: Which chapter contains this beat
  • Scene: Optional specific scene
  • Tension: Tension level at this point (1-5)
  • Notes: Author notes about this beat

3. Tension Curve

Visual editor for the emotional intensity across your story. Points on the curve define tension levels at different story positions (0.0-1.0).

4. Subplots

Secondary story threads with:

  • Name: Subplot identifier
  • Type: romance, rivalry, mystery, growth, revenge, mentorship
  • Characters: Who is involved
  • Start Beat: Where it begins
  • Resolve Beat: Where it concludes
  • Touchpoints: Where it intersects main plot

Templates Tab

The Templates tab provides reusable pattern templates that can be instantiated in scenes.

Template Structure

Templates have:

  • Variables: Placeholders for entities ($hero, $location, etc.)
  • Body: Sequence of statements using variables
  • Bindings: How variables map to actual entities when instantiated

Example Template

"Mentor's Wisdom" is pattern
"Mentor's Wisdom" has variable $student as character
"Mentor's Wisdom" has variable $mentor as character
"Mentor's Wisdom" has variable $lesson as theme

"Mentor's Wisdom" has template begin
  $mentor meets $student
  $mentor reveals $lesson
  $student transforms "gains understanding"
"Mentor's Wisdom" has template end

NL Tab

The NL (Natural Language) tab generates prose from your CNL specification.

Generation Options

  • Create Story: Generate complete prose narrative from specification
  • Style: Literary style preferences
  • Length: Target word count or detail level
  • Focus: Specific chapters/scenes to generate

Output

The NL tab displays:

  • Generated prose text
  • Word count and statistics
  • Quality metrics for generated content

CNL Tab

The CNL tab shows the raw CNL specification generated by the visual editor.

Features

  • View: Read-only display of generated CNL
  • Export: Download CNL as .cnl or .txt file
  • Copy: Copy to clipboard
  • Syntax Highlighting: Keywords and structure highlighted

Workflow

Visual Editor → CNL (auto-generated) → Parser → AST → Metrics

Authors don't typically write CNL directly - they use the visual editor, which generates CNL automatically.
